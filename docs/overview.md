## 1. 目的

* 画像（スクリーンショット・写真・手書き含む）から料理レシピを自動抽出し、構造化データ（JSON）として保存・閲覧できる機能の実証。
* OCR と LLM を組み合わせたレシピ構造化ワークフローの技術的妥当性を検証する。

## 2. ワークフロー
1. ユーザーがブラウザで CookScan にアクセス。
2. 画像をドラッグ & ドロップでアップロード。
3. バックエンドが OCR (Gemini 2.5 Pro) → LLM 構文解析 (GPT‑4o) を実行。
4. JSON 形式でレシピが生成され、画面にプレビュー表示。
5. 必要ならユーザーが材料名・数量・手順を編集。
6. 保存ボタンでローカル DB に書き込み、一覧に追加。


## 3. 技術スタック

* 画像処理: LLM(Gemini)
* LLM: OpenAI(GPT-4o)
* フロントエンド: React+Vite
* バックエンド: Hono
* データベース: SQLite
